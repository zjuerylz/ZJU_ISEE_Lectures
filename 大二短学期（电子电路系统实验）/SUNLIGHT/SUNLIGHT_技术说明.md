# SUNLIGHT 技术说明

## 1. 系统功能

本工程用于太阳能充电、储能电容管理和 LED 恒流控制，主要功能包括：

- 根据太阳能充电电流判断白天或黑夜。
- 白天关闭 LED，并根据 VBAT 状态进行充电。
- 黑夜停止充电，并在电池保护允许时驱动 LED。
- LED 电流支持 0～300 mA 设置，步进为 25 mA。
- OLED 显示运行模式、VBAT、日夜状态、充电状态及 LED/充电参数。
- 充电通路与 LED 通路互锁，避免两条功率通路同时导通。

## 2. 主要信号

| MCU 引脚 | 信号 | 功能 |
| --- | --- | --- |
| PA0 | ADCVBAT | 采集 VBAT 电压 |
| PA2 | ADCILED | 采集 LED 电流 |
| PA3 | ADCICHARGE | 采集充电电流，并用于 DAY/NGT 判断 |
| PB0 / TIM3_CH3 | 充电控制 | 通过 C8T6 控制并联分流 MOS Q3 |
| PB1 / TIM3_CH4 | LED 控制 | 反相控制 LED MOS |
| PB5～PB8 | S1～S4 | 模式、定时和电流调节按键 |
| PB10 / PB11 | I2C2 | OLED 的 SCL/SDA |
| PC13 | 状态灯 | 正常时呼吸，故障时快速闪烁 |

PA1/ADCVSL 已从运行代码和 CubeMX ADC 配置中移除。

PB0 为高电平时，C8T6 导通并拉低 Q3 栅极，Q3 关断，允许太阳能向 VBAT 充电。PB0 为低电平时，Q3 导通并分流太阳能输入，停止充电。

## 3. 白天和黑夜判断

日夜判断只使用 ADCICHARGE，不再使用 ADCVSL：

1. 关闭 LED，并使充电与 LED 功率通路进入安全状态。
2. 关断 Q3，使太阳能输入能够向 VBAT 提供电流。
3. 若此前处于分流状态，等待输入源恢复，并持续检查 VBAT 过压。
4. 对 PA3 的 ADCICHARGE 连续采样并计算充电电流。
5. 充电电流不小于 50 mA时判定为 `DAY`，否则判定为 `NGT`。

上电后会尽快执行首次检测。正常情况下，白天约每 5 秒复检一次；黑夜或充电完成状态约每 30 秒复检一次。检测期间 LED 保持关闭。

## 4. 充电控制

充电状态包括：

- `IDLE`：等待满足充电条件。
- `BULK`：以较高占空比充电。
- `CV`：随 VBAT 上升逐步降低有效充电占空比。
- `REST`：关闭充电后测量静置电压。
- `DONE`：充电完成。

主要控制条件：

- VBAT 不高于 7.8 V 时进入 BULK，目标有效充电占空比为 90%。
- VBAT 达到 7.8 V 后进入 CV，并在 7.8～8.3 V 范围内逐步降低充电占空比。
- VBAT 达到 8.45 V 时立即关闭充电并进行静置确认。
- VBAT 不低于 8.2 V、充电电流持续低于 100 mA 约 5 秒后进入静置确认。
- 静置电压满足完成条件后进入 `DONE`；电压下降到重启门限后重新充电。

OLED 上的充电占空比表示最终“允许充电”的有效比例，不是单纯的 PB0 电平比例。充电采用分段导通方式，使外部限流电源和储能电容有足够的恢复时间。

## 5. LED 控制

LED 默认目标电流为 300 mA，可在 0～300 mA 范围内调节：

- S3：减小 25 mA。
- S4：增加 25 mA。
- 0 mA：保持 LED 关闭。

上电后 300 mA 目标采用直接导通；第一次调节电流后进入 PA2 电流闭环。程序根据目标值和实测电流逐步调节反相 PWM，占空比显示采用 LED 实际导通比例。

VBAT 低于 6.0 V 时关闭 LED；VBAT 恢复到 6.4 V以上并稳定后重新允许 LED 工作。无论 AUTO、定时还是手动模式，白天状态都会禁止 LED 输出。

## 6. 按键功能

| 按键 | 功能 |
| --- | --- |
| S1 | 在 AUTO、ON、OFF 三种模式之间切换 |
| S2 | 请求定时点亮，持续约 10 秒 |
| S3 | LED 目标电流减小 25 mA；0 mA 后回到 300 mA |
| S4 | LED 目标电流增加 25 mA；300 mA 后回到 0 mA |

所有 LED 点亮请求仍受日夜判定、VBAT 低压保护、ADC 状态和功率通路互锁限制。

## 7. OLED 显示

OLED 每 500 ms 刷新一次。开机初始化时会短暂显示 `SUNLIGHT R12`，进入主循环后固定显示四行运行信息。

### 7.1 第一行：版本、工作模式、LED 实际状态和运行时间

显示格式：

```text
R12 <MODE> <LED><SSSS>
```

例如：

```text
R12 AUTO ON 0123
R12 ON OFF0456
R12 TIME ON 0789
```

各字段含义如下：

| 显示符号 | 含义 |
| --- | --- |
| `R12` | 当前固件功能版本标识。 |
| `AUTO` | 自动日夜模式。检测为 `DAY` 时请求关闭 LED；检测为 `NGT` 时请求点亮 LED。是否真正点亮仍受 VBAT 低压保护、ADC 状态、充电静置和功率通路互锁限制。 |
| `ON`（模式位置） | 手动点亮请求模式，由 S1 切换得到。它只表示“请求 LED 点亮”，不能绕过白天禁亮、低压保护、ADC 故障或功率通路保护。 |
| `OFF`（模式位置） | 手动关闭模式，不再请求 LED 点亮。 |
| `TIME` | S2 启动的约 10 秒定时点亮请求。计时结束后自动进入 `OFF` 模式。该请求同样不能绕过白天禁亮和其他保护。 |
| `ON`（模式后的状态位置） | LED MOS 当前已经产生物理输出，LED 通路实际处于导通状态。 |
| `OFF`（模式后的状态位置） | LED 当前没有物理输出。即使模式为 `ON` 或 `TIME`，也可能因为白天、低电压、日夜状态未知、ADC 故障、充电静置或通路切换而显示 `OFF`。 |
| `SSSS` | 系统上电后的累计运行秒数，显示范围为 `0000`～`9999`，超过后从 `0000` 重新循环。它不是 TIME 模式的剩余时间。LED 状态为 `OFF` 时，实际屏幕会紧接着显示数字，例如 `OFF0456`。 |

因此，`R12 ON OFF0456` 并不矛盾：第一个 `ON` 表示软件正在请求点亮，第二个 `OFF` 表示保护逻辑尚未允许 LED 真正输出。

S1 每按一次依次切换：

```text
AUTO → ON → OFF → AUTO
```

### 7.2 第二行：VBAT 电压和 LED 低压保护许可

显示格式：

```text
BAT x.xxV ON
BAT x.xxV OFF
```

| 显示符号 | 含义 |
| --- | --- |
| `BAT` | 储能电容或电池端的 VBAT 电压。数值经过 ADC 分压换算、实测校准和滤波，以伏为单位显示两位小数。 |
| `V` | 电压单位伏特。 |
| `ON` | VBAT 低压保护已经放行，电压条件允许 LED 工作；这不代表 LED 此刻一定点亮。 |
| `OFF` | VBAT 低压保护未放行，LED 必须关闭。VBAT 低于 6.0 V 时立即进入该状态。 |

从 `OFF` 恢复到 `ON` 需要 VBAT 高于 6.4 V 并连续稳定约 1 秒。6.0～6.4 V 之间保持原状态，形成滞回，避免 LED 在门限附近频繁开关。

注意：第二行的 `ON/OFF` 表示“电池电压是否允许 LED 工作”，第一行的 `ON/OFF` 表示“LED 是否真的在输出”，两者含义不同。

### 7.3 第三行：日夜判断和充电状态

显示格式：

```text
SOL DAY BULK
SOL NGT IDLE
SOL --- IDLE
```

| 显示符号 | 含义 |
| --- | --- |
| `SOL` | 太阳能输入及充电控制模块标识。当前版本不再显示 SOLAR 电压。 |
| `DAY` | 判定为白天。程序在释放 Q3 后检测到 ADCICHARGE 不小于 50 mA。白天禁止 LED 输出，并允许充电状态机根据 VBAT 决定是否充电。 |
| `NGT` | `NIGHT` 的缩写，表示判定为黑夜。检测到的充电电流小于 50 mA，此时停止太阳能充电，并允许 LED 按模式和保护条件工作。 |
| `---` | 日夜状态尚未得到有效检测结果，常见于刚上电、正在重新检测、ADC 数据无效或功率通路尚未稳定。状态未知时 LED 不会点亮，正常充电控制也不会直接放行。 |
| `IDLE` | 充电空闲或等待状态。当前没有进入 BULK/CV 充电，可能正在等待有效白天状态、合适的 VBAT 或重新启动条件。 |
| `BULK` | 大电流充电阶段。VBAT 较低时采用较高的有效充电占空比。 |
| `CV` | 恒压调节阶段。随着 VBAT 接近上限，程序逐步降低有效充电占空比。 |
| `REST` | 静置确认阶段。程序暂时停止充电，等待端电压稳定后判断是否真正充满。 |
| `DONE` | 充电完成状态。充电保持关闭，直到 VBAT 下降并稳定达到重新充电条件。 |

如果编译时关闭太阳能充电功能，第三行会显示 `SOL DISABLED`；正常交付版本启用了太阳能充电。

### 7.4 第四行：LED 参数或充电参数

第四行会根据当前日夜及功率通路状态在两种格式之间切换。

#### LED 参数格式

```text
Lxxx/yyy Pzzz%
```

例如：

```text
L288/300 P100%
```

| 显示符号 | 含义 |
| --- | --- |
| `L` | LED 电流信息标识。 |
| `xxx` | PA2/ADCILED 测得并经过校准、滤波后的 LED 实际电流，单位为 mA。 |
| `/` | 分隔实际电流和目标电流。 |
| `yyy` | 用户设定的 LED 目标电流，范围为 0～300 mA，S3 步减、S4 步进，每次改变 25 mA。 |
| `P` | LED 的物理导通占空比。由于 PB1 后级为反相驱动，程序已经换算成“LED 实际导通比例”后再显示。 |
| `zzz%` | LED 导通占空比，范围为 0%～100%。`P100%` 表示 LED MOS 已达到最大连续导通；它不保证实际电流一定达到目标值，实际电流还会受 VBAT、LED 压降、MOS、二极管和接线压降影响。 |

夜晚时即使低压保护使 LED 关闭，第四行仍可能显示该格式，此时通常会看到实际电流接近 0 mA、`P0%`。

#### 充电参数格式

```text
Dxxx% CyyymA
```

例如：

```text
D 90% C185mA
```

| 显示符号 | 含义 |
| --- | --- |
| `D` | 充电“允许导通”的有效占空比。 |
| `xxx%` | 充电状态机请求的有效充电比例。它表示最终允许太阳能向 VBAT 充电的时间比例，不是 PB0 原始高低电平比例，也不是某一瞬间 Q3 的状态。 |
| `C` | 充电电流标识。 |
| `yyy` | PA3/ADCICHARGE 测得并经过滤波后的平均充电电流，单位为 mA。 |
| `mA` | 电流单位毫安。 |

`D0%` 只表示当前没有正常充电导通时间，可能原因包括黑夜、`IDLE`、`REST`、`DONE`、日夜检测、VBAT 保护或功率通路互锁，并不一定表示故障。

### 7.5 异常显示

```text
ADC ERROR
```

表示至少一个必要 ADC 通道没有得到有效数据。程序会进入安全状态，关闭正常充电输出和 LED 输出，避免使用错误测量值继续控制功率通路。

第三行不再显示 SOLAR 电压，也不再显示旧版本中的 `S`、`S*` 或 ADCVSL 诊断数据。

## 8. 安全机制

- 充电通路和 LED 通路切换时先进入两路关闭状态，再等待安全死区。
- ADC 读取失败时关闭充电和 LED。
- 日夜状态未知时关闭 LED，并暂停正常充电控制。
- 太阳能探测期间持续检查 VBAT，避免探测过程造成过压。
- OLED I2C 异常时执行总线恢复，不改变功率通路保护逻辑。
- 系统错误时 Q3进入分流状态，同时关闭 LED。
